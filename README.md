# cnsite-kaiyuan-lab

本项目是一个用于归档和发布多个独立 HTML 页面的仓库。所收录的页面均为静态 HTML 文件，不依赖于任何特定域名或外部服务，旨在提供一种轻量、可复用的页面存档与展示方式。

## 项目简介

在开发或学习过程中，常常会积累一些独立的 HTML 页面——可能是原型演示、功能测试页、信息展示页，或是小型工具页面。本仓库将这些页面集中存放，便于统一管理、版本控制与公开访问。

页面内容保持原始结构，不添加额外跟踪、统计或营销代码。所有页面均可通过 GitHub Pages 等静态托管服务直接浏览。

## 目录说明

```
.
├── index.html          # 仓库入口页面（可选），列出所有归档页面链接
├── pages/              # 归档的 HTML 页面存放目录
│   ├── example-page-1/
│   ├── example-page-2/
│   └── ...
├── assets/             # 公共资源（CSS、JS、图片等），按需使用
├── README.md           # 本文件
└── LICENSE             # 许可证文件
```

- `pages/`：每个子目录对应一个独立页面，包含该页面所需的全部 HTML、CSS、JavaScript 及资源文件，确保可独立运行。
- `assets/`：如需多个页面共用样式或脚本，可放置于此处，并在页面中引用。

## 页面归档说明

每个归档页面应满足以下要求：

- 页面完整自包含：单个 HTML 文件或一个独立目录（包含所有依赖资源）。
- 页面文件命名清晰，目录名称与页面主题对应。
- 页面内不包含任何外部不可控资源（如第三方 CDN、外部 API 调用），以保证长期可用性。
- 页面中可包含简单的说明或标题，方便浏览者理解内容。

如需新增页面，请在 `pages/` 下新建目录，并将相关文件放入其中。同时建议在 `index.html` 中添加链接。

## 维护说明

- 本仓库由维护者不定期更新，欢迎通过 Issue 提出建议或报告问题。
- 提交新页面时，请确保文件完整、无外部依赖，并注明页面用途。
- 本仓库仅用于归档静态 HTML 页面，不提供任何形式的托管承诺或服务保证。
- 所有页面内容遵循仓库所用许可证（见 `LICENSE` 文件），除非页面内另有声明。

## 使用建议

- 可直接通过 GitHub Pages 将本仓库部署为静态站点，浏览所有归档页面。
- 也可将页面文件下载到本地，离线查看或二次修改。
- 如需引用本仓库中的页面，请保留页面内的原始作者或来源信息（如有）。
